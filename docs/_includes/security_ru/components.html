Компоненты trdl и их функции
  • Vault

    Vault — система управления секретами и шифрованием на основе аутентификации и авторизации. Обеспечивает безопасное хранение секретов (сертификатов, паролей, ключей API и др.), шифрование, ротацию и отзыв ключей, аудит и не только.

    Vault защищает от многих видов внутренних и внешних угроз, актуальных для системы управления секретами.

    • Безопасное управление ключами шифрования.
    • Платформа для безопасного запуска trdl-сервера, которая обеспечивает конфиденциальность, целостность и доступность данных, отчетность и аутентификацию.
  • TUF-репозиторий

    Доверенное хранилище, организованное на основе TUF (The Update Framework). Фреймворк обеспечивает безопасное хранение обновлений, а также их безопасное скачивание.

    TUF-репозиторий защищает контент от различных атак: «заморозки», установки стороннего ПО, скачивания ненужных зависимостей и т. д.

    • Защита от несанкционированного доступа к контенту: бинарным файлам, пакетам и другому софту.
    • Актуальность, согласованность и целостность данных.
    • Защита от компрометации и потери ключей; снижение ущерба в случае их компрометации и потери.
    • Восстановление ключей.
  • Git-репозиторий

    Система контроля версий в облаке — например, GitHub или GitLab.

    • Хранилище кода и конфигурации.
    • GPG-подписи для верификации операций (release / publish).
  • Docker

    ПО для контейнеризации приложений.

    • Воспроизводимость сборки.
