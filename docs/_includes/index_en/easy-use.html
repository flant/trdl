Easy-to-use
For developers
You don't need any additional knowledge or tools to use trdl:
  • If your build runs in Docker, it will run in trdl as well. All you need to do is specify the image name and the build command;
  • The only requirement is that the assembly results have to be stored in a specific directory;
  • There are ready-made GitHub Actions for essential workflows.
For users
You can add the package source to the host in just a single command. trdl makes life easier; e.g., it allows the user to:
  • automatically receive the latest software version from the release channel;
  • manually pull and activate the newest software version from the release channel;
  • activate it within the current bash session;
  • use playbooks and other automation tools.
