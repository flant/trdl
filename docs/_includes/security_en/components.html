trdl components and their features
  • Vault

    Vault is a secret and encryption management system based on authentication and authorization. It enables secure storage of secrets (certificates, passwords, API keys, etc.) and provides encryption services, key rotation/revocation, auditing, and more.

    Vault protects your secret management system against many types of internal and external threats.

    • Manages encryption keys securely.
    • Safely runs a trdl server to provide confidentiality, integrity, availability, accountability, authentication.
  • TUF repository

    TUF repo is a trusted repository based on TUF (The Update Framework). The framework ensures secure update storage and distribution.

    The TUF-based repository protects the content against various attack types: indefinite freeze, arbitrary installation, extraneous dependencies attacks, and so on.

    • Protects against unauthorized access to content: binary files, packages, and other software.
    • Ensures freshness, consistency, and integrity of data.
    • Mitigates the risk of key loss; provides compromise resilience (damage reduction in case of key compromise/loss)
    • Replaces keys.
  • Git repository

    A cloud-based version control system in the cloud, e.g., GitHub or GitLab.

    • Stores code and configuration.
    • Signs release/publish actions with GPG.
  • Docker

    A software for packaging applications into containers.

    • Ensures builds are reproducible.
